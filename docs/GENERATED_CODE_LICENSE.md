# Licensing of Melange-Generated Code

## Summary

Code generated by `melange compile` is licensed under the **GNU General Public
License v3.0 or later** (GPL-3.0-or-later), the same license as Melange itself.

## Why generated code is a derived work

Melange's code generator does not perform a mechanical, non-creative
transformation of user input (like a compiler translating C to assembly).
Instead, the generated output **incorporates substantial copyrighted material**
from the Melange project:

- **Device model equations** — Shockley diode, Ebers-Moll/Gummel-Poon BJT,
  Koren triode plate current, Shichman-Hodges JFET, Level 1 MOSFET, and
  Blackmer/THAT VCA models are emitted verbatim from GPL-licensed Tera
  templates (`crates/melange-solver/templates/rust/device_*.rs.tera`).

- **Newton-Raphson solver logic** — The nonlinear convergence loop, SPICE-style
  voltage limiting (pnjlim/fetlim), damping heuristics, backward Euler
  fallback, and chord method optimizations are all emitted from GPL templates.

- **Matrix operations** — Sparse LU factorization, matrix-vector products, and
  the DK/Schur complement solver structure come directly from GPL source.

- **Numerical infrastructure** — `safe_exp`, `fast_exp`, denormal flushing,
  NaN recovery, DC blocking filters, and oversampling polyphase filters are
  all copied from GPL-licensed Melange primitives and templates.

A typical generated file contains 500–2000 lines of code, of which the vast
majority is drawn directly from these GPL-licensed sources. The user's circuit
netlist contributes only the matrix constants and topology — the algorithmic
code is Melange's.

This is analogous to how Bison (the parser generator) generates code that
includes substantial portions of the Bison runtime, making the output a
derived work of Bison — which is why Bison includes a special exception for
its output. **Melange does not include such an exception.**

## What this means in practice

- You **may** use melange-generated code in any project, commercial or
  otherwise, provided you comply with the GPL-3.0-or-later.

- If you **distribute** a binary (plugin, application, library) that contains
  melange-generated code, you must make the corresponding source code
  available under GPL-3.0-or-later.

- You **may** use melange-generated code internally (e.g., for testing,
  prototyping, internal tools) without any distribution obligation.

- You **may** sell GPL-licensed plugins. The GPL does not prohibit commercial
  use — it requires that recipients receive the source code and the same
  freedoms.

## Rationale

Melange exists because of the open-source community. The device models,
solver algorithms, and validation infrastructure represent significant
engineering effort contributed under copyleft terms. Requiring that
derivative works remain open ensures that improvements flow back to the
community rather than being captured by closed-source products.

If a company wants to build plugins with Melange, they are welcome to — but
they stand on the shoulders of open giants, and the GPL ensures they
contribute back.
